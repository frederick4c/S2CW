\section*{2. Priors}
\subsection*{(a) Qualitative Comparison}
% Limit: Under 200 words.
The plain order statistics prior involves drawing $k$ change points from a uniform distribution over the interval $[0, L]$ and then sorting them. This means that the points are independent before sorting and can be clustered tightly together. As shown in \Cref{fig:2a}, the marginal distributions are broad and can peak directly at the boundaries.

The even-numbered prior is constructed by drawing $2k + 1$ points uniformly, sorting them, and then taking the evenly indexed points. This results in a repulsive effect between points, encouraging more evenly spaced change points. It also has the effect of preventing peaks at the boundaries, which makes sense for a change point, as the first and last points are always odd and therefore always excluded. In \Cref{fig:2a} we can also see that the marginal distributions are more distinctly separated and narrower than those of the plain order priors.

\subsection*{(b) PDF Derivation}
% Requirement: Show the PDF of the even-numbered order statistics.
To derive the even-numbered pdf we start with the fact that the plain ordered pdf for $2k+1$ points on the unit interval $[0, 1]$ is a constant
$$P(u_{(1)}, u_{(2)}, \dots, u_{(2k+1)}) = (2k + 1)!$$

This is valid only on the strictly ordered region $0 < u_{(1)} < u_{(2)} < \dots < u_{(2k+1)} < 1$.

For the even numbered pdf we only keep even-numbered order statistics $\hat{s}_j = u_{(2j)}$ so must marginalise the plain order pdf over all the other discarded points. We know that the points are ordered and on the unit interval $[0, 1]$ so the integral is
$$P(\hat{s}_1, \dots, \hat{s}_k) = \int_{0}^{\hat{s}_1} \int_{\hat{s}_1}^{\hat{s}_2} \dots \int_{\hat{s}_k}^{1} (2k + 1)! \, du_{(2k+1)} \dots du_{(3)} du_{(1)}$$

Because the integrand $(2k+1)!$ is a constant, each integral becomes the length of the interval between the 2 points on the bounds, or the gap $\hat{g}_j$ between them.
$$P(\hat{s}_1, \dots, \hat{s}_k) = (2k + 1)! (\hat{s}_1)(\hat{s}_2 - \hat{s}_1)\dots(1 - \hat{s}_k)$$

$$=  (2k + 1)! \prod_{j=1}^{k+1} \hat{g}_j$$

We now need to map this from the unit interval $[0, 1]$ to the actual timeline $[0, L]$. We are given the scaling relationship $s_j = L\hat{s}_j$ so can scale the gaps according to $\hat{g}_j = g_j / L$ and substitute this into the equation
$$P(\hat{s}_1, \dots, \hat{s}_k) = (2k + 1)! \prod_{j=1}^{k+1} \left( \frac{g_j}{L} \right)$$

Since $L$ is a constant it can be pulled out of the product
$$P(\hat{s}_1, \dots, \hat{s}_k) = \frac{(2k + 1)!}{L^{k+1}} \prod_{j=1}^{k+1} \left( g_j \right)$$

We must then apply the change of variables formula 
$$P(s_1, \dots, s_k) = P(\hat{s}_1, \dots, \hat{s}_k) \times \left| J \right|$$

where $|J|$ is the Jacobian determinant of the transformation. 

Since $s_j = L \hat{s}_j$, the inverse transformation is $\hat{s}_j = \frac{s_j}{L}$. Taking the derivative of each variable gives $\frac{\partial \hat{s}_j}{\partial s_j} = \frac{1}{L}$. The Jacobian determinant is then given by the product of the derivatives and since there are $k$ points,
$$\left| J \right| = \left( \frac{1}{L} \right)^k = \frac{1}{L^k}$$

This allows us to obtain the final pdf
$$\pi(s_1, s_2, \dots, s_k | M_k) = \frac{(2k+1)!}{L^{2k+1}} \prod_{j=1}^{k+1} g_j$$

$$\pi(s_1, s_2, \dots, s_k | M_k) \propto \prod_{j=1}^{k+1} g_j$$

\section*{3. The constant rate model, $M_0$}
% Requirement: Prior vs Posterior plot and evidence calculation.
From the instructions we know that the log likelihood is given by 
$$\log \mathcal{L}(\{I_i\} | \{s_j\}, \{h_j\}, M_k) = \sum_{i=1}^{N} \log h_0 - \int_{0}^{L} dt \, x(t)$$

Since $x(t) = h_0$ is a constant over the interval $L$ 
$$\log \mathcal{L}(\{I_i\} | h_0, M_0) = N \log(h_0) - h_0 L$$ 

The likelihood is therefore 
$$\mathcal{L}(\{I_i\} | h_0, M_0) = h_0^N \exp(-h_0 L)$$

From Exercise 2, the prior for the height parameters is universally given by a Gamma distribution $h_0 \sim \Gamma(\alpha, \beta)$ where $\alpha=1$ and $\beta=200$. This forms the probability density function
$$ \pi(h_0|M_0) = \frac{\beta^\alpha}{\Gamma(\alpha)} h_0^{\alpha-1} \exp(-\beta h_0) $$ 

Since $\alpha=1$ and $\Gamma(1) = 1$, the prior simplifies to 
$$ \pi(h_0|M_0) = \beta \exp(-\beta h_0) = 200 \exp(-200 h_0) $$

Applying Bayes' rule the unnormalised posterior $P^*$ is the product of the prior and likelihood
$$ P^*(h_0 | \{I_i\}, M_0) = \beta \cdot h_0^N \exp(-h_0(L + \beta)) $$

To calculate the true evidence $Z_0$ for model $M_0$, we must integrate $P^*(h_0 | \{I_i\}, M_0)$ over all possible values of $h_0$.
$$Z_0 = \int_{0}^{\infty} P^*(h_0 | \{I_i\}, M_0) \, dh_0$$

When evaluating $P^*(h_0) = \beta \cdot h_0^N \exp(-h_0(L+\beta))$ for our dataset where $N=191$ and $L=40550$, the probabilities evaluate to 0 due to an effect known as floating point underflow.

To safely resolve this underflow, we perform the computation in log-space. First, we take the logarithm of the unnormalised posterior:
$$f(h_0) = \log(P^*(h_0)) = \log(\beta) + N \log(h_0) - h_0(L + \beta)$$

We can then find the exact maximum of this log-posterior analytically. Taking the derivative with respect to $h_0$ and setting to zero yields the maximum a posteriori (MAP) height:
$$h_{\text{max}} = \frac{N}{L + \beta} \approx 0.00469$$

By evaluating the maximum log-posterior $C = f(h_{\text{max}})$, we can shift the posterior to peak at 1 and prevent the underflow problem when calculating the evidence.
$$P^*(h_0) = \exp(C) \times \exp(f(h_0) - C)$$

$$Z_0 = \exp(C) \int_{0}^{\infty} \exp(f(h_0) - C) \, dh_0$$

We can now use numerical integration with \texttt{scipy.integrate.quad} to evaluate the integral
$$\log Z_0 = C + \log\left( \int_{0}^{\infty} \exp(f(h_0) - C) \, dh_0 \right) \approx -1217$$

The prior and posterior distributions are plotted in \Cref{fig:3a}.

\subsection*{(a) MCMC Analysis}
% Briefly explain (in under 300 words) which algorithm/package you used for the MCMC, any important settings for this algorithm, and any diagnostic checks you performed to check for convergence.
For the MCMC analysis, we use the \texttt{emcee} package which implements an affine invariant ensemble sampler. It works by creating an ensemble of multiple walkers that propose their next moves based on the position of the other walkers in the ensemble. This makes them invariant to the scale of the parameters, which is important given that $s_1$ is in the order of $10^4$ and $h_0$,$h_1$ are in the order of $10^{-3}$.

There are 3 dimensions to search over $[s_1, h_0, h_1]$ so 32 walkers were used to ensure full coverage. From the trace plots in \Cref{fig:4a_traces} we can see that the chains converged very quickly and that the burn-in period of 500 steps was sufficient to avoid skewing results. This fast convergence is likely due to the fact that the walkers were initialised at values close to the peaks for each parameter, chosen from the results of $M_0$. The trace plots each remain stable within a flat, horizontal band suggesting that they reached a stationary distribution. The autocorrelation times ($\tau$) for the chains were calculated to be $44 < \tau < 52$ meaning that the 8000 steps were easily enough to obtain a large number of independent samples. The chains were not thinned as the sample size is too small to be concerned with memory usage. The Gelman-Rubin statistic was not calculated as the walkers look at other walkers to determine their next step, meaning they are inherently not independent.

The corner plot in \Cref{fig:4a_corner} shows that the $s_1$ parameter has 3 peaks and that a single change point may not accurately describe the data.

\subsection*{(b) Savage Dickey Density Ratio}
% Explain briefly (in under 200 words) how the Savage Dickey density ratio can, in principle, be used to find the ratio of evidences between the models 𝑀1 and 𝑀0.
% Also explain what difficulty is encountered when trying to use the Savage Dickey density ratio in this example. include plot
The Savage-Dickey Density Ratio (SDDR) is a method for calculating the evidence ratio of two nested models with consistent priors. Since $M_0$ can be interpreted as a special case of $M_1$ where $h_0 = h_1$, the models are nested and the priors on the parameters $h$ are consistent as they given by the same Gamma distribution. This means that the SDDR can, in principle, be used to find the evidence ratio of $M_1$ to $M_0$.

We calculate the SDDR by the ratio of the posterior and prior density under the nested condition $\Delta h = h_0 - h_1 = 0$.
$$B_{01} = \frac{P(\Delta h = 0 | D, M_1)}{\pi(\Delta h = 0 | M_1)}$$

The prior density can be calculated analytically
$$\pi(\Delta h) = \frac{\beta}{2} \exp(-\beta |\Delta h|) = \frac{200}{2} \exp(0) = 100$$

The problem arises in this example when trying to evaluate the posterior density at $\Delta h = 0$. From \Cref{fig:4a_corner}, we can see that the marginalised posteriors for $h_0$ and $h_1$ are tightly constrained around 0.0085 and 0.0026 respectively, meaning there will be very little overlap and very few, if any, MCMC samples with $\Delta h = 0$. As a result, we can see that the kernel density estimate for the posterior difference in \Cref{fig:4b} peaks at $\approx 0.006$ and has effectively 0 density at $\Delta h = 0$. This will result in an SDDR $B_{01}$ of 0 and $\log B_{01} \to -\infty$.

\subsection*{(c) Laplace Approximation}
% Explain briefly (in under 200 words) what difficulty is encountered if one tried to use the Laplace approximation to estimate the evidence 𝑍1 ≡ 𝑃({𝐼𝑖}|𝑀1) for the model 𝑀1
The Laplace approximation is a semi analytic technique for calculating the evidence which approximates the posterior as a Gaussian around its peak. For $M_1$ there are two issues that prevent us from using this approximation effectively.
$$\log Z \approx \log P(D | \theta_{MAP}, M) + \log \pi(\theta_{MAP} | M) + \frac{d}{2} \log(2\pi) - \frac{1}{2} \log |H|$$

The main issue is that the Laplace approximation requires the posterior to be smooth as it calculates the Hessian matrix $H$. As $s_1$ varies continuously, the log likelihood changes discontinuously each time $s_1$ passes through an accident time $y_i$, because the rate assigned to that accident jumps between $h_0$ and $h_1$. This will result in a second derivative with respect to $s_1$ that is undefined at $y_i$ and 0 everywhere else. The determinant of the Hessian $|H|$ will therefore equal 0 making the $-\frac{1}{2} \log(0)$ term mathematically undefined.

The second issue is that the posterior is multimodal, as can be seen in \Cref{fig:4a_corner} where the marginal posterior for $s_1$ has 3 distinct peaks. Fitting a single Gaussian around the main peak would ignore an significant proportion of the distribution, leading to an underestimation even if the Hessian was well defined.

\subsection*{(d) Nested Sampling}
% Describe briefly (in under 200 words) the details of the algorithm and any relevant settings used.
To calculate the evidence $Z$ we use the nested sampling package \texttt{dynesty}. Nested sampling works by initialising a set of live points and then iteratively replacing the points of lowest likelihood with one of higher likelihood. By keeping track of how the volume of the prior space changes the algorithm can numerically integrate the posterior to find $Z$.

We used 1000 live points to ensure a high resolution for numerical integration with a default stopping criterion of $\delta \log Z = 0.1$. Design choices to improve the efficiency of the algorithm included setting \texttt{bound='multi'} rather than the default single ellipsoid. The use of multiple bounds is important as we have seen in \Cref{fig:4a_corner} that the posterior is multimodal. Another choice was to use \texttt{sample='rwalk'} method for sampling, which is well-suited to lower dimensional multimodal problems.

The calculated evidence for $M_1$ was $\log{Z_1} = -1118.53 \pm 0.10$ resulting in a log Bayes factor $\log B_{10} \approx 28.56$ and the Log Posterior Odds $\approx 29.66$. This suggests that $M_1$ is mathematically superior to $M_0$.

\subsection*{(c) Model Averaging}
% Briefly explain (in under 250 words) how you calculated the uncertainty regions
After discarding the first 10\% of samples as burn in (based on the trace plot), the remaining RJMCMC samples were used for model averaging. The autocorrelation time was estimated to be $\tau \approx 195$, giving a moderate effective sample size (ESS) of approximately 1384 so the estimates may be slightly noisy. The chain was not thinned as memory was not an issue. 

To calculate the uncertainty regions, we define a grid of 500 evenly spaced time points over $[0,L]$. For each retained RJMCMC sample we evaluate $x(t)$ for each point in the grid, producing a $270,000 \times 500$ matrix where each row is an MCMC iteration and every column is a specific point in time.

Since RJMCMC samples models in proportion to their posterior, evaluating the distribution of $x(t)$ values at each time step (down the column) naturally marginalises over the number of change points $k$. We can then construct the median rate and the 50\% and 90\% credible intervals at each time point shown in \Cref{fig:5c} by calculating the 5th, 25th, 50th, 75th, and 95th percentiles. Despite independently calculating the marginal uncertainty at each time point, the curve is smooth as it averages over many piecewise models. The final results suggest a strong decrease around 1890 with smaller variations around 1930 and 1945, roughly the time of the second world war.
